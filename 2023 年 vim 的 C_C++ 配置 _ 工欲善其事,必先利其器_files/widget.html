Nice article! Respect and marked.
记录并和大家分享一下我在Ubuntu22.04上面按照教程配置中教程没有详细叙述的步骤：
Ubuntu22.04安装LSP(Language Server Protocol ):

  1. marksman
  2. efm
  3. ccls
sudo apt update
apt list --upgradable
sudo apt upgrade
sudo apt -y install efm-langserver
sudo snap install marksman
sudo apt -y install ccls
sudo apt install npm
sudo apt -y install cargo
sudo apt -y install xclip
sudo apt install lua5.4

mv ~/.config/nvim ~/.config/nvim.bak
cd ~
export https_proxy=http://127.0.0.1:7890 && export http_proxy=http://127.0.0.1:7890 && export all_proxy=socks5://127.0.0.1:7890
git clone --depth=1 https://github.com/martins3/My-Linux-config .dotfiles
ln -s ~/.dotfiles/nvim ~/.config/nvim
nvim
## 在 nvim 中间执行 checkhealth 命令
## 在 nvim 中间执行 :Lazy build rsync.nvim命令

用nvim打开一个.c或者.cpp文件，在neovim中执行命令：
：LspInfo
查看Lsp状态。
下面是怎么使用ccls建立一个项目，参考https://github.com/MaskRay/ccls/wiki/LanguageClient-neovim 

cd ~/.config/nvim
vim setting.json

在setting.json中写入以下内容：

{
  "initializationOptions": {
    "cache": {
      "directory": "/home/t/NutstoreFiles/Nutstore/cppcodes/ccls"
    }
  }
}

vim ~/.config/nvim/init.vim

在init.vim中写入以下内容：

let g:LanguageClient_serverCommands = {
    \ 'c': ['ccls', '--log-file=/cppcodes/cc.log'],
    \ 'cpp': ['ccls', '--log-file=/cppcodes/cc.log'],
    \ 'cuda': ['ccls', '--log-file=/cppcodes/cc.log'],
    \ 'objc': ['ccls', '--log-file=/cppcodes/cc.log'],
    \ }

let g:LanguageClient_loadSettings = 1 " Use an absolute configuration path if you want system-wide settings
let g:LanguageClient_settingsPath = '/home/t/.config/nvim/settings.json'
" https://github.com/autozimu/LanguageClient-neovim/issues/379 LSP snippet is not supported
"let g:LanguageClient_hasSnippetSupport = 0

配置ccls的项目建立的第一个例子：

cd /home/t/NutstoreFiles/Nutstore/cppcodes; mkdir c; cd c
git init  # let LanguageClient-neovim know this is a project
echo '#include <stddef.h>\nint main() {}' > a.cc
nvim a.cc
# :LanguageClientStart

For a larger project, you'll need .ccls or compile_commands.json in your project root.
